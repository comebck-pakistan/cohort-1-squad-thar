## 💡 Our Idea

Our squad picked **[Idea 3 — The Shoebox-to-Books Bookkeeper](https://github.com/Comebck-Pakistan/cohort-1-product-challenge#idea-3--the-shoebox-to-books-bookkeeper)** from the [Cohort 1 Product Challenge](https://github.com/Comebck-Pakistan/cohort-1-product-challenge). Below are our ideation-stage validation answers, reproduced exactly as we submitted them.

### One specific user

a small restraunt owner receiving online payment confirmation details through whatsapp

### The painful problem

the problem is that the payment details for a small bussiness owner are scattered on different platforms be it whatsapp hard form receipts and screenshots

the owner will switch bcz all the payment details will gather at one place with needing any financial or accounting literacy

### Our 8-week wedge

A digital accountant with all records of payments done, payments to be made along with remainders

### The one number that proves people want it

i reach out one of my friend he is in marketing specifically B to B and he has hired an accountant for all this stuff i talked to him and told him about my idea and he said that it would be great if he can get such service

### Why we picked this idea

Yes i can definitely reach out 10 real users and i myself have observed that my friend got a subscription of some xyz tool and he forgot to unsubscribe it for the next month and it costed him around $200. That was one real problem most of the users face they forget to unsubscribe the subscriptions or where they have added their payment details so if they can get a remainder before the next payment is due then it would solve their major problem

## 🚀 Getting Started — Clone & Contribute

Follow these steps to get this repo on your machine and start contributing code.

### 1. Prerequisites
- Install [Git](https://git-scm.com/downloads)
- A [GitHub account](https://github.com/join) added to the **Comebck-Pakistan** organization
- _(Recommended)_ [VS Code](https://code.visualstudio.com/) and/or the
  [GitHub CLI](https://cli.github.com/)

### 2. Clone the repository
```bash
git clone https://github.com/Comebck-Pakistan/cohort-1-squad-thar.git
cd cohort-1-squad-thar
```

### 3. Create a branch for your work
Never commit straight to `main`. Create a branch for each feature or fix:
```bash
git checkout -b your-name/short-description
```

### 4. Make your changes, then stage and commit them
```bash
git add .
git commit -m "Briefly describe what you changed"
```

### 5. Push your branch to GitHub
```bash
git push -u origin your-name/short-description
```

### 6. Open a Pull Request
- Open the repo on GitHub and click **"Compare & pull request"**.
- Describe your change, request a review from a squadmate, and merge once approved.

### 🔄 Keep your local copy up to date
Before starting new work, pull the latest `main`:
```bash
git checkout main
git pull origin main
```
